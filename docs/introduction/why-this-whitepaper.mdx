## What This Whitepaper Is

This is a **reverse engineering analysis** of Anthropic's officially released **Claude Code CLI**.

The source code has been processed through decompilation (TypeScript single-file bundle reverse engineering), retaining core functional modules but containing numerous `unknown`/`never`/`{}` type errors — these don't affect Bun runtime execution, but it means our analysis is based on runtime behavior + residual source code structure, not original source code.

**This is NOT:**
- Official documentation or usage tutorial
- API reference manual
- A feature pitch for Claude Code

**This IS:**
- An architectural deconstruction of a production-grade agentic system
- The "why" behind each design decision
- Reusable engineering patterns: agentic loop, tool abstraction, context engineering, defense-in-depth security

## Most Elegant Design Decisions Discovered During Reverse Engineering

### 1. Agentic Loop's Self-Healing Capability

The core loop implemented in `src/query.ts` is not a simple "send request -> receive response." It is a **self-healing state machine**:

- API returns error (rate limiting, token exceeded) -> auto retry/degrade
- Tool execution timeout -> backgrounding + notification mechanism
- Conversation too long triggers compaction -> compress history and seamlessly continue
- User interruption -> generates `UserInterruptionMessage` so AI understands what happened

This is not "if-else stacking" but rather letting AI decide the next step based on context — even when unexpected things happen.

### 2. Context Engineering's Layered Strategy

AI doesn't have true "memory." Claude Code creates this illusion through careful layering:

| Layer | Mechanism | Persistence |
|-------|-----------|-------------|
| **System Prompt** | Project structure, git status, CLAUDE.md | Rebuilt each turn |
| **Conversation History** | Complete User/Assistant/Tool messages | Within session |
| **Compaction** | Auto-compress overly long conversations into summaries | Replaces original messages after compression |
| **Memory Files** | Cross-session persistent notes | Permanent (user-controlled) |
| **File History** | File modification timestamp snapshots | Within session |

When `src/context.ts` assembles the System Prompt, the strategy is: **static content first, dynamic content last** — this leverages the API's caching mechanism, allowing cached tokens to be reused when the prefix remains unchanged.

### 3. Tool System's Dual-Track Permission Model

`src/tools/BashTool/shouldUseSandbox.ts` demonstrates an elegant dual security model:

- **Application layer**: Permission rules determine "can it execute" (allowlist/blocklist/user confirmation)
- **OS layer**: Sandbox determines "what it can do during execution" (file system/network/process isolation)

The trust assumptions differ between layers: the application layer trusts user configuration, while the OS layer trusts nothing. Even if AI bypasses application-level permissions (theoretically impossible, but defense in depth), the OS-level sandbox still limits actual damage.

### 4. Feature Flags as Global Switch

A single line of code in `src/entrypoints/cli.tsx` determines the entire system's behavior:

```typescript
const feature = (_name: string) => false;
```

All `feature('FLAG_NAME')` calls return `false` — this means Anthropic's internal experimental features (COORDINATOR_MODE, KAIROS, PROACTIVE, etc.) are all disabled. In official builds, these flags are injected at compile time through Bun's `bun:bundle`, with different user groups seeing different features.

This is a **progressive release architecture**: the same codebase controls feature visibility through feature flags, without maintaining multiple branches.

### 5. Compaction's Tiered Strategy

`src/services/compact/` implements three compression strategies:

- **Micro-compact**: When a single tool output is too long, truncate the result
- **Auto-compact**: When conversation tokens approach the limit, automatically compress history
- **Reactive-compact**: When API returns a token exceeded error, emergency compress and retry

This isn't simply "chopping old messages" — it uses AI itself to summarize previous conversation, preserving semantic information. After compression, a `TombstoneMessage` is inserted to mark the boundary.
