## What Is feature()

Claude Code uses the Bun bundler's `bun:bundle` module to provide compile-time feature gating:

```typescript
// Usage in source code (src/tools.ts, etc.)
import { feature } from 'bun:bundle'

const SleepTool = feature('PROACTIVE') || feature('KAIROS')
  ? require('./tools/SleepTool/SleepTool.js').SleepTool
  : null
```

In Anthropic's internal builds, `feature()` is evaluated at bundle time — code where it returns `true` is retained, and code where it returns `false` is completely removed by **Dead Code Elimination (DCE)**.

In our decompiled version, this function is stubbed as:

```typescript
// src/entrypoints/cli.tsx line 3
const feature = (_name: string) => false;
```

This means all code behind the 88+ feature flags **will never execute at runtime**, but the code itself is fully preserved and can be read and analyzed.

## Typical Patterns in Code

Feature flags are used in three main patterns in the codebase:

### Pattern 1: Conditional Tool Loading

```typescript
// src/tools.ts — The most common pattern
const MonitorTool = feature('MONITOR_TOOL')
  ? require('./tools/MonitorTool/MonitorTool.js').MonitorTool
  : null
```

When the flag is `false`, the `require()` call is removed by DCE, and the tool does not appear in the available tools list.

### Pattern 2: Conditional Command Registration

```typescript
// src/commands.ts — Register slash commands
if (feature('VOICE_MODE')) {
  commands.push({ name: 'voice', description: '...' })
}
```

### Pattern 3: Conditional API Feature Enablement

```typescript
// src/constants/betas.ts — Control beta headers sent to the API
export const AFK_MODE_BETA_HEADER = feature('TRANSCRIPT_CLASSIFIER')
  ? 'afk-mode-2026-01-31'
  : ''
```

<Note>
Since `feature()` is evaluated at build time, code removed by DCE does not increase the final bundle size. However, in the decompiled version, all this code is preserved — which is precisely why we can perform a complete analysis.
</Note>

## Interesting Findings

- **The KAIROS family** is the largest — 6 related flags control everything from core functionality to push notifications
- **ABLATION_BASELINE** is used for "scientific control experiments" — it disables thinking, compaction, auto-memory, and other advanced features to measure the baseline performance of raw API calls
- **BUDDY** is an AI mascot/sprite system — with a complete implementation in the `src/buddy/` directory
- **ULTRAPLAN** and **ULTRATHINK** hint at reasoning modes more advanced than the current extended thinking
